An Introduction to Bayesian Data Analysis for Cognitive Science

2.1 Probability

Informally, we all understand what the term ``probability’’ means. We routinely talk about the probability of events like it raining today. However, there are two distinct ways to think about probability. One can think of probability from a frequency-of-occurrence perspective; here, the probability of an event is defined with respect to the frequency with which it might occur in repeated observations. Such a conception of probability is easy to imagine in cases where an event can, at least in principle, occur repeatedly. An example would be obtaining a 6 when tossing a die again and again. However, such a conception is difficult to justify when talking about certain one-of-a-kind events, such as earthquakes. In such situations, probability is best seen as expressing our uncertainty about the event happening. In this book, depending on our particular goal in a given situation, we will need both these conceptions of probability.

Regardless of whether it is seen from the frequency or the uncertainty perspective, probability of an event happening is defined to be constrained in the following way:

  • The probability of an event must lie between 0 and 1, where 0 means that the event is impossible and cannot happen, and 1 means that the event is certain to happen.
  • For any two mutually exclusive events, the probability that one or the other occurs is the sum of their individual probabilities.
  • Two events are independent if and only if the probability of both events happening is equal to the product of the probabilities of each event happening.
  • The probabilities of all possible events in the entire sample space must sum up to 1.

In the context of data analysis, we will talk about probability in the following way. Consider some data that you might have collected. This could be discrete 0,1 responses in a question-response accuracy task, continuous measurements of reading times in milliseconds from an eyetracking study, etc. We will say that the data are being generated from a so-called random variable, which we will designate with a capital letter such as \(Y\) (we could have used any letter). The actually observed data will be distinguished from the random variable generating it by using lower case \(y\). The random variable Y will always have a probability distribution \(p(y)\) associated with it; this distribution \(p(y)\) assigns a probability to sets of possible realizations of \(Y\). The distribution will generally have a parameter or set of parameters \(\theta\) associated with it; for a given set of data \(y\), we can write \(p(y|\theta)\) to represent the probability distribution. The vertical bar expresses a conditional statement: the probability distribution of \(y\) given a specific value of \(\theta\). The parameter \(\theta\) is assumed to have some fixed but unknown value. A notational convention: when \(\theta\) represents a set of \(p\) parameters, we will write \(\theta=<\theta_1,\dots,\theta_p>\); in that case, the assumption is that each \(\theta_i\), where \(i = 1, \dots, p\) has a fixed but unknown value. In the discussion below, for simplicity we will just assume that we are talking about a single parameter \(\theta\).

In frequentist approaches to data analysis, the observed data \(y\) are used to draw inferences about \(\theta\). A typical question that we ask in the frequentist paradigm: does \(\theta\) have a particular value \(\theta_0\)? One can obtain estimates of the unknown value of \(\theta\) from the observed data \(y\), and then draw inferences about how different–or more precisely how far away–this estimate is from the hypothesized \(\theta_0\). This is the essence of null hypothesis significance testing. The usual conclusions are framed in terms of rejecting the hypothesis that \(\theta\) has value \(\theta_0\), or failing to reject this hypothesis.

Bayesians begin a different question. What is common to the frequentist paradigm is that we assume that the data are generated from a random variable \(Y\) and that there is a probability distribution \(p(y|\theta)\) indexed by the parameter \(\theta\). Where the Bayesian approach diverges from the frequentist is that the goal is to express our uncertainty about \(\theta\). In other words, we treat the parameter \(\theta\) itself as a random variable, which means that we assign a probability distribution \(p(\theta)\) to this random variable. This distribution \(p(\theta)\) is called the prior distribution on \(\theta\); later on, we will spend some time trying to understand how such a distribution is defined for a range of different research problems. Given such a prior distribution and some data \(y\), the end-product of a Bayesian data analysis is the so-called posterior distribution of the parameter given the data: \(p(\theta | y)\). This posterior distribution is the probability distribution of \(\theta\) after conditioning on \(y\).

2.1.1 Conditional probability and deriving Bayes’ rule

Suppose that A stands for the event the streets are wet'' and B stands forit has been raining.’’ We can talk about the probability of the streets being wet given that it has rained; or more generally, the probability of A given that B has happened. This kind of statement is written as \(Prob(A|B)\) or more simply \(P(A|B)\). This is the conditional probability of event A given B. Conditional probability is defined as follows.

\[\begin{equation} [#eq:condprob] P(A|B)= \frac{P(A,B)}{P(B)} \hbox{ where } P(B)>0 \end{equation}\]

We can rearrange equation B.2 (??) so that we can talk about the joint probability of both events A and B happening: \(P(A,B)=P(A|B)P(B)\).

Now, since the probability A and B happening is the same as the probability of B and A happening, i.e., since \(P(B,A)=P(A,B)\), we can

\[\begin{equation} P(A,B) = P(A|B) P(B) = P(B,A) = P(B|A)P(A) \end{equation}\]

The above equality allows us to state that

\[\begin{equation} P(A|B) P(B) = P(B|A)P(A) \end{equation}\]

Dividing both sides by P(B):

\[\begin{equation} P(A|B)=\frac{P(B|A)P(A)}{P(B)} \end{equation}\]

The above statement is Bayes’ rule, and is the basis for all the statistical inference we will do in this book. For now, this is all the probability theory we need to know!
